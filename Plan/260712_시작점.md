HTML 기반으로 플레이어블 게임 메이커를 만들려고해. 새로운 기획서에 따라서 게임을 개발하게 되는데, 매번 처음부터 새로 시작하는 것도 불편하고, 기존에 이미 구축된 사례를 참고하면 더 빠르게 구현이 가능할 것 같아. 예를 들면, 플레이어 움직임 제어 로직, 적 AI 움직임 제어 로직 등등 다양한 로직들이 모듈화 처리되면 향후 지속적으로 만들어지는 게임들이 모두 빠르게 구현될 수 있을것 같아.

주로 클로드 코드나 코덱스를 활용해서 HTML 기반 게임을 만들게 될거야. 어떻게 기반을 구축하고 재활용할 수 있을지, 어떻게 하면 효율을 높일 수 있을지 상세 계획을 수립해줘. 대략 아래와 같은 내용들이 있어야 AI와 대화하면서 리소스를 채워넣거나 UI/UX를 수정하거나, 구현된 로직들을 모듈화해서 저장해놓을 수 있을것 같아.
