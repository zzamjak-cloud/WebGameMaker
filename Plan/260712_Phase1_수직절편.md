# WebGameMaker Phase 1 수직 절편 계획

- 작성일: 2026-07-12
- 기준: [260712_구축계획.md](./260712_구축계획.md)
- 작업 브랜치: `codex/phase-1-topdown-slice`

## 1. 목표

최소 game/scene/entity JSON을 입력으로 실제 한 판을 완주할 수 있는 탑다운 액션 샘플을 만든다. 플레이어 이동, 적 순찰·추적, 조사광 공격, 체력·접촉 피해, 승패, React HUD, 재시작을 포함하고 모듈 추출 전 재사용 비용을 고정한다.

완료 게이트:

- 기본 `/`에서 시작 → 플레이 → 승리 또는 패배 → 재시작을 완주한다.
- `pnpm e2e --grep @vertical-slice`가 Chromium에서 승리·패배 두 경로와 console error 0을 검증한다.
- 기존 `?view=compat`의 Chromium·Firefox·WebKit 호환성, opaque iframe, destroy/recreate 잔존 0 검증을 유지한다.
- `tests/benchmarks/reuse-baseline.json`이 게임별 gameplay LOC와 수동 설정 단계 기준선을 재현한다.

## 2. 게임 방향

제목은 **수문 07 — 마지막 등불**이다. 폐쇄 직전의 해안 수문에서 정비 드론이 조사광으로 먹물 생물을 몰아내고 세 신호등을 다시 켠다.

핵심 루프:

1. `WASD` 또는 방향키로 이동한다.
2. 적은 평소 정해진 구간을 순찰하고 감지 거리 안에서 플레이어를 추적한다.
3. `Space` 조사광 펄스로 가까운 적에게 피해를 준다.
4. 적을 제거할 때마다 신호등 하나가 켜진다.
5. 신호등 3개를 모두 켜면 승리하고, 선체 체력이 0이면 패배한다.
6. 결과 오버레이의 `R` 또는 버튼으로 같은 초기 상태에서 다시 시작한다.

조사광 열은 공격할 때 오르고 자동으로 식는다. 100%에서 잠시 과열되며, 게임 세계의 빛 부채꼴과 HUD 열 게이지가 같은 상태를 표현한다.

## 3. 시각·HUD 기준

1970년대 해안 수문 안전 표지, 산화 금속, 조수표와 프레넬 렌즈를 시각 언어로 사용한다. 기존 밝은 계측대 UI를 확장하지 않고 실제 게임 화면은 16:9 전체 viewport로 구성한다.

색 토큰:

- `harbor-950 #071A21`: 바다와 배경
- `tide-700 #1D4650`: 수문 구조물
- `foam-100 #E6EFE7`: 기본 글자와 물보라
- `brass-400 #E3B85B`: 목표와 열 게이지
- `signal-500 #E7563F`: 피해와 위험
- `lumen-300 #74E0C2`: 조사광과 회복

```text
┌──────────────────────────────────────────────────────────────┐
│ 선체 82/100       ◉──◉──○ 신호등 2/3       렌즈 열 64% ◔   │
│                                                              │
│            Phaser 월드 · 조사광 · 플레이어 · 적              │
│                                                              │
│ F3 제작 정보             SPACE 조사광             먹물체 01  │
└──────────────────────────────────────────────────────────────┘
```

- Phaser는 월드, 물리, 조사광과 엔티티를 그린다.
- React DOM은 체력, 목표, 열, 남은 적, 조작 안내와 결과 오버레이를 그린다.
- HUD는 색 외에 숫자, 채움, 아이콘 형태를 함께 사용한다.
- `F3`는 상태값과 HUD safe-area/anchor를 표시하는 제작 정보 오버레이다.
- 결과 오버레이는 DOM 버튼, 초점 이동, `aria-modal`, `aria-live`를 제공한다.
- `prefers-reduced-motion`에서는 흔들림, 맥동, 빠른 섬광을 제거한다.

## 4. 코드 경계

```text
games/floodgate-07/
├── game.project.json
├── scenes/main.scene.json
├── reuse-recipe.json
└── src/
    ├── compileProject.ts
    ├── gameState.ts
    └── features/
        ├── playerMovement.ts
        ├── enemyAi.ts
        └── combat.ts

apps/player/src/
├── App.tsx
├── vertical-slice/
│   ├── VerticalSliceView.tsx
│   ├── verticalSliceRuntime.ts
│   └── verticalSliceTypes.ts
└── game/
    └── playerRuntime.ts
```

실행 경계:

```text
JSON bundle → 기존 schema validator → 게임 전용 compileProject
            → TopDownScene → Phaser 월드
            → session snapshot → React HUD
```

- game/scene JSON은 저장소의 단일 원본이다.
- `compileProject.ts`는 필요한 `moduleId`와 config를 좁히는 게임 전용 adapter만 둔다.
- 범용 module registry, lifecycle, capability/event bus와 Studio 인스펙터는 Phase 2/5로 미룬다.
- 게임별 gameplay 코드는 `games/floodgate-07/src/features`에 두고 아직 공용 패키지로 승격하지 않는다.
- 기존 Phase 0 controller와 자원 정리 경계를 재사용하며 호환성 화면은 `?view=compat`로 보존한다.

## 5. 상태 모델

```text
phase: playing | won | lost
player: health/maxHealth, position, facing, invulnerableUntil
lamp: heat, overheated
enemies: id, health, mode(patrol|chase|dead), position
objective: litBeacons, totalBeacons
run: runId, elapsedMs
```

- 적 감지 거리와 추적 이탈 거리를 분리해 AI 상태 떨림을 막는다.
- 접촉 피해에는 무적 시간을 두어 한 번의 overlap이 여러 피해로 중복되지 않게 한다.
- 승패가 결정되면 Phaser 물리 입력을 멈추고 React 결과 오버레이만 활성화한다.
- 재시작은 scene 상태, HUD 상태, 입력과 timer를 초기값으로 되돌린다.

## 6. 검증

단위 테스트:

- patrol → chase → return 전환
- 체력 clamp, 접촉 피해 무적 시간, 조사광 열·과열·냉각
- 적 제거와 신호등 진행, 승패 판정, restart 초기화
- JSON module config compile 성공·오류

E2E:

- 실제 keyboard 입력으로 이동과 조사광 공격을 수행한다.
- 적이 patrol에서 chase로 전환하고 HUD 수치가 session snapshot과 일치하는지 확인한다.
- 세 적을 실제 공격 경로로 제거해 승리 → 결과 오버레이 → 재시작을 확인한다.
- 별도 실행에서 접촉 피해로 패배 → 결과 오버레이 → 재시작을 확인한다.
- 재시작 3회 후 canvas/listener/timer 잔존 0과 console error 0을 확인한다.
- 테스트 bridge는 snapshot과 결정적 clock 보조만 제공하며 teleport, 강제 피해, 강제 승패 API는 만들지 않는다.

재사용 비용:

- `tests/benchmarks/measure-reuse.ts`가 TypeScript scanner로 `games/*/src/features/**/*.ts`의 trivia·주석을 제외한 실제 코드 행을 센다.
- `reuse-recipe.json`의 실행 가능한 수동 단계 배열 길이를 `manualSetupSteps`로 센다.
- 파일·recipe SHA-256을 함께 저장하고 timestamp는 넣지 않아 같은 입력에서 byte-for-byte 같은 결과를 만든다.
- `pnpm measure:reuse -- --check`가 `tests/benchmarks/reuse-baseline.json`과 drift를 검사하며, 갱신은 명시적 `--write-baseline`에서만 허용한다.
